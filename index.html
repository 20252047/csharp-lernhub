C# String & Exception Referenz

HTL POS1 — Vollständige Methodenreferenz mit try-catch

⚠️ Das wichtigste Konzept: Immutability

Strings sind unveränderlich (immutable). Jede Methode erzeugt ein neues String-Objekt. Der alte String bleibt unverändert — Ergebnis immer zuweisen!

string s = "Hallo";
s.Replace("Hallo", "Welt");       // ❌ Ergebnis wird verworfen!
s = s.Replace("Hallo", "Welt");  // ✅ immer zuweisen!

📐 Eigenschaften (keine Klammern!)

s.Length Eigenschaft → int
Wann: Immer wenn du Zeichen zählst — für Schleifen, Substring, Bedingungen. KEINE Klammern!
string s = "Hallo";
Console.WriteLine(s.Length);       // → 5
Console.WriteLine(s.Length - 1);  // → 4 (Index des letzten Zeichens)
❌ Häufiger Fehler: s.Length() mit Klammern → Compilerfehler!
s[index] Indexer → char
Wann: Einzelne Zeichen lesen. Schreiben auf den String direkt ist verboten!
string s = "Hallo";
char erstes = s[0];              // → 'H'
char letztes = s[s.Length - 1]; // → 'o'
// ❌ Verboten — String ist schreibgeschützt:
s[0] = 'X';  // → Compilerfehler!

🔍 Such-Methoden

s.IndexOf(wert)→ int
Erstes Vorkommen finden. Gibt -1 zurück wenn nicht gefunden!
string s = "animal";
int pos = s.IndexOf("n");  // → 1
int pos2 = s.IndexOf("z"); // → -1
if (pos != -1) { // ✅ immer prüfen! }
s.LastIndexOf(wert)→ int
Letztes Vorkommen — sucht von hinten. Gibt -1 wenn nicht gefunden.
string pfad = @"c:\folder\file.txt";
int pos = pfad.LastIndexOf('.'); // → 15
string ext = pfad.Substring(pos); // → ".txt"
s.Contains(wert)→ bool
Prüft ob ein Zeichen/String vorkommt.
string s = "The quick brown fox";
Console.WriteLine(s.Contains("fox")); // True
Console.WriteLine(s.Contains('z'));   // False
StartsWith / EndsWith→ bool
Anfang/Ende prüfen — URLs, Dateiendungen, Validierung.
string url = "www.google.com";
Console.WriteLine(url.StartsWith("www.")); // True
Console.WriteLine(url.EndsWith(".com"));  // True

✂️ Extraktions-Methoden

s.Substring(startIndex)→ string — ab Index bis Ende
Alles ab einer bestimmten Position bis zum Ende holen.
string s = "c# is what I love and what I love is c#";
string teil = s.Substring(27);
Console.WriteLine(teil); // → "what I love is c#"
s.Substring(startIndex, länge)→ string — genau N Zeichen
Genau N Zeichen ab einer Position ausschneiden.
⚠️ 2. Parameter ist die LÄNGE, NICHT der Endindex!
string s = "Hallo Welt";
string teil = s.Substring(6, 4);
Console.WriteLine(teil); // → "Welt" (ab Index 6, 4 Zeichen)

// Klassisch: Dateiname ohne Extension
string pfad = @"c:\folder\myFile.txt";
int start  = pfad.LastIndexOf('\\') + 1;
int punkt  = pfad.LastIndexOf('.');
string name = pfad.Substring(start, punkt - start); // → "myFile"
s.ToCharArray()→ char[]
Beschreibbares char-Array erzeugen — wenn du einzelne Zeichen verändern musst.
string s = "AaBbCcDd";
char[] chars = s.ToCharArray();
for (int i = 0; i < chars.Length; i++)
    Console.WriteLine(i + ": " + chars[i]);
chars[0] = 'x'; // ✅ char[] ist beschreibbar!
s.Split(trennzeichen)→ string[]
String bei einem Trennzeichen in Teile aufteilen — häufigste Test-Methode!
string csv = "Mathe;Deutsch;Englisch;POS1";
string[] faecher = csv.Split(';');
// faecher[0]="Mathe", [1]="Deutsch", ...
foreach (string f in faecher) Console.WriteLine(f);

// Mehrere Trennzeichen:
string[] teile = text.Split(new char[] { ',', ';', ' ' });

✏️ Änderungs-Methoden (immer Ergebnis zuweisen!)

s.Insert(index, wert)→ string
Fügt einen String VOR dem angegebenen Index ein.
✅ Insert-Index: 0 = Anfang | Length = Ende (KEIN Length-1 fürs Ende!)
string s = "aaabbb";
string result = s.Insert(3, " ");
Console.WriteLine(result); // → "aaa bbb"

string r2 = s.Insert(0, ">>> ");   // → ">>> aaabbb"
string r3 = s.Insert(s.Length, "!"); // → "aaabbb!" (Length, nicht Length-1!)
Warum Length und nicht Length-1? Insert fügt VOR dem Index ein. Index Length ist die Position NACH dem letzten Zeichen.
s.Remove(startIndex, anzahl)→ string
Löscht eine bestimmte Anzahl Zeichen ab dem angegebenen Index.
string s = "Hello World";
string r = s.Remove(5, 6); // ab Index 5, 6 Zeichen weg
Console.WriteLine(r);      // → "Hello"

// Extension entfernen:
string datei = "myFile.txt";
int punkt = datei.LastIndexOf('.');
string ohneExt = datei.Remove(punkt, datei.Length - punkt);
Console.WriteLine(ohneExt); // → "myFile"
s.Replace(alt, neu)→ string
Ersetzt ALLE Vorkommen eines Zeichens/Strings durch einen neuen Wert.
// String → String
string s = "This docment uses docments";
s = s.Replace("docment", "document");
Console.WriteLine(s); // → "This document uses documents"

// char → char
string csv = "1 2 3 4 5";
csv = csv.Replace(' ', ',');
Console.WriteLine(csv); // → "1,2,3,4,5"
s.Trim()→ string
Entfernt führende und nachfolgende Leerzeichen. Immer vor TryParse und nach Eingaben!
string s = "  Hallo Welt  ";
Console.WriteLine(s.Trim());       // → "Hallo Welt"
Console.WriteLine(s.TrimStart()); // → "Hallo Welt  "
Console.WriteLine(s.TrimEnd());   // → "  Hallo Welt"
s.ToUpper()→ string
string s = "Hallo";
Console.WriteLine(s.ToUpper()); // → "HALLO"
s.ToLower()→ string
string s = "HALLO";
Console.WriteLine(s.ToLower()); // → "hallo"
if (s.ToLower() == "ja") { } // case-insensitiv

🛡️ Statische Prüf-Methoden

string.IsNullOrEmpty(s)static→ bool
true wenn null ODER "" (leer). Aufruf ohne Instanz!
Console.WriteLine(string.IsNullOrEmpty(""));    // True
Console.WriteLine(string.IsNullOrEmpty(null));  // True
Console.WriteLine(string.IsNullOrEmpty("Hi")); // False
string.IsNullOrWhiteSpace(s)static→ bool
true wenn null, "" ODER nur Leerzeichen. Sicherste Methode!
Console.WriteLine(string.IsNullOrWhiteSpace("   ")); // True
Console.WriteLine(string.IsNullOrWhiteSpace(""));    // True
Console.WriteLine(string.IsNullOrWhiteSpace(null));  // True
String.EmptyFeld→ string
Leerstring-Initialisierung — bevorzugt gegenüber "".
string s = String.Empty;  // entspricht ""
string s2 = "";           // auch gültig

🚨 Exception Handling (try-catch-finally)

Eine Exception ist ein Programmzustand der zu einem Fehler führt. Mit try-catch fängst du ihn ab statt das Programm abstürzen zu lassen.

try
{
    // Code der einen Fehler werfen könnte
}
catch (Exception ex)
{
    Console.WriteLine("Fehler: " + ex.Message);
}
finally
{
    // wird IMMER ausgeführt — auch bei Fehler
    // typisch: Verbindungen schließen, Ressourcen freigeben
}

Die wichtigsten Exceptions für String & Collections

IndexOutOfRangeException

Wann: Du greifst auf einen Index zu der nicht existiert — z.B. s[10] bei Length 5.
string s = "Hi"; // Length = 2, Indizes: 0 und 1
try
{
    char c = s[5]; // ❌ IndexOutOfRangeException!
}
catch (IndexOutOfRangeException ex)
{
    Console.WriteLine("Index zu groß: " + ex.Message);
}
// ✅ Vorher prüfen:
if (index >= 0 && index < s.Length)
{
    char c = s[index];
}

ArgumentOutOfRangeException

Wann: Substring, Remove, Insert mit ungültigem Index oder Länge aufgerufen.
string s = "Hallo"; // Length = 5
try
{
    string teil = s.Substring(3, 99); // ❌ zu lang!
}
catch (ArgumentOutOfRangeException ex)
{
    Console.WriteLine("Ungültiger Bereich: " + ex.Message);
}
// ✅ Sicher:
if (start + len <= s.Length)
    string teil = s.Substring(start, len);

FormatException

Wann: int.Parse("abc") — String kann nicht in Zahl umgewandelt werden. Lösung: TryParse!
try
{
    int zahl = int.Parse("abc"); // ❌ FormatException!
}
catch (FormatException ex)
{
    Console.WriteLine("Keine gültige Zahl: " + ex.Message);
}
// ✅ BESSER: TryParse — kein try-catch nötig!
if (int.TryParse("abc", out int zahl))
    Console.WriteLine("Zahl: " + zahl);
else
    Console.WriteLine("Keine gültige Zahl!");

DivideByZeroException

Wann: Division durch 0 — z.B. beim Durchschnitt berechnen wenn Count == 0.
try { int e = 10 / 0; } // ❌
catch (DivideByZeroException)
    Console.WriteLine("Division durch 0!");

// ✅ Vorher prüfen:
if (liste.Count > 0)
    double schnitt = summe / liste.Count;

NullReferenceException

Wann: Du rufst eine Methode auf einem null-String auf.
string s = null;
try { int len = s.Length; } // ❌ NullReferenceException!
catch (NullReferenceException)
    Console.WriteLine("String ist null!");

// ✅ Vorher prüfen:
if (!string.IsNullOrWhiteSpace(s))
    int len = s.Length;

ArgumentException (Dictionary)

Wann: Dictionary.Add() mit einem Key der schon existiert.
Dictionary<string, int> d = new Dictionary<string, int>();
d.Add("Anna", 1);
try
{
    d.Add("Anna", 2); // ❌ ArgumentException! Key existiert schon
}
catch (ArgumentException)
    Console.WriteLine("Key existiert bereits!");

// ✅ ContainsKey vorher prüfen:
if (!d.ContainsKey("Anna")) d.Add("Anna", 2);
else d["Anna"]++;

📦 Generic Collections — System.Collections.Generic

Warum Collections statt Arrays? Arrays haben eine fixe Größe. Collections sind dynamisch — sie wachsen/schrumpfen automatisch.

Generic <T>: T ist ein Platzhalter für den Datentyp. List<int> akzeptiert nur int — Typfehler werden beim Compilieren erkannt.

CollectionPrinzipWann verwenden?
List<T>Dynamisches ArrayGeordnete Liste, Index-Zugriff, variable Größe
Dictionary<TK,TV>Schlüssel → Wert (Hash)Blitzschnelles Nachschlagen per eindeutigem Key
SortedList<TK,TV>Wie Dictionary, aber sortiertAutomatisch nach Key sortiert
Stack<T>LIFO — Last In, First OutVerlauf, Rückgängig, Klammer-Check
Queue<T>FIFO — First In, First OutDruckerwarteschlange, Ticket-System

1. List<T> — Die dynamische Liste

Zugriff per Index wie beim Array, aber dynamisch wachsend. Statt .Length heißt es hier .Count!

// Variante 1: Objekt-Initialisierer
List<int> liste = new List<int>() { 1, 2, 3, 4, 5 };
// Variante 2: Leer, dann befüllen
List<string> namen = new List<string>();
Add(item)→ void
Element am Ende hinzufügen.
liste.Add(99);    // → [1,2,3,4,5,99]
namen.Add("Anna");
Insert(index, item)→ void
An einem bestimmten Index einfügen.
liste.Insert(0, 0); // → [0,1,2,3,4,5,99]
Remove(item)→ bool
Erstes Vorkommen entfernen.
liste.Remove(3);  // → true, entfernt 3
liste.Remove(99); // → false, nicht vorhanden
RemoveAt(index)→ void
Element an einem bestimmten Index entfernen.
liste.RemoveAt(0); // entfernt erstes Element
// ❌ ArgumentOutOfRangeException wenn Index zu groß!
Sort()→ void
liste.Sort();    // [0,1,2,4,5,99]
namen.Sort();   // alphabetisch
Contains / IndexOf / Count
liste.Contains(5);  // → true/false
liste.IndexOf(5);   // → Index oder -1
Console.WriteLine(liste.Count);

foreach — Standard-Iteration

foreach (string name in namen)
    Console.WriteLine(name);

// Mit Index (for-Schleife):
for (int i = 0; i < liste.Count; i++)
    Console.WriteLine(i + ": " + liste[i]);

2. Dictionary<TKey, TValue>

Blitzschneller Key-Value-Lookup. Jeder Key ist einzigartig. Reihenfolge ist nicht garantiert!

Dictionary<int, string> dict = new Dictionary<int, string>()
{
    { 1, "Huber" },
    { 2, "Mayer" },
    { 3, "Wimmer" }
};
Add(key, value)→ void
dict.Add(4, "Schmidt");
// ❌ ArgumentException wenn Key schon da!
if (!dict.ContainsKey(4))
    dict.Add(4, "Schmidt");
TryGetValue(key, out val)→ bool
Sicherer Zugriff ohne Exception bei fehlendem Key!
if (dict.TryGetValue(1, out string name))
    Console.WriteLine(name); // → "Huber"
else
    Console.WriteLine("Key nicht gefunden");

foreach mit KeyValuePair

foreach (KeyValuePair<int, string> eintrag in dict)
    Console.WriteLine(eintrag.Key + " → " + eintrag.Value);

// Kurz mit var:
foreach (var e in dict)
    Console.WriteLine(e.Key + " → " + e.Value);

3. SortedList<TKey, TValue>

Wie Dictionary, aber Einträge werden automatisch nach Key sortiert gehalten. Alle Dict-Methoden gelten auch hier, plus IndexOfKey() und RemoveAt(index).

SortedList<int, string> sl = new SortedList<int, string>();
sl.Add(3, "C"); sl.Add(1, "A"); sl.Add(2, "B");
// Automatisch sortiert: {1:A, 2:B, 3:C}

int idx = sl.IndexOfKey(2); // → 1
sl.RemoveAt(0);             // entfernt Key 1 ("A")

4. Stack<T> — LIFO: Last In, First Out

Wie ein Stapel Teller: Das zuletzt Abgelegte wird zuerst genommen. Pushing = drauflegen, Popping = wegnehmen.

Stack<string> verlauf = new Stack<string>();
Push(item)→ void
Element oben drauflegen.
verlauf.Push("Seite1");
verlauf.Push("Seite2");
verlauf.Push("Seite3");
Pop()→ T
Oberstes Element holen UND entfernen. Exception wenn leer!
if (verlauf.Count > 0)
    Console.WriteLine(verlauf.Pop());
// → "Seite3"
Peek()→ T
Oberstes Element NUR ansehen — NICHT entfernen.
if (verlauf.Count > 0)
    Console.WriteLine(verlauf.Peek());
// → "Seite2" (bleibt da)
while-Schleife
while (verlauf.Count > 0)
{
    Console.WriteLine(verlauf.Pop());
}
// → Seite2, Seite1 (LIFO!)

InvalidOperationException — Stack leer

Wann: Pop() oder Peek() auf leerem Stack aufgerufen.
Stack<int> s = new Stack<int>();
try
{
    int x = s.Pop(); // ❌ InvalidOperationException!
}
catch (InvalidOperationException)
    Console.WriteLine("Stack ist leer!");

// ✅ Vorher prüfen:
if (s.Count > 0) Console.WriteLine(s.Pop());

5. Queue<T> — FIFO: First In, First Out

Wie eine Warteschlange beim Arzt: Wer zuerst kommt, wird zuerst verarbeitet. Hinten rein, vorne raus.

Queue<string> drucker = new Queue<string>();
Enqueue(item)→ void
Element hinten einreihen.
drucker.Enqueue("Dokument_A.pdf");
drucker.Enqueue("Dokument_B.pdf");
drucker.Enqueue("Dokument_C.pdf");
Dequeue()→ T
Vorderstes Element holen UND entfernen. Exception wenn leer!
if (drucker.Count > 0)
    Console.WriteLine(drucker.Dequeue());
// → "Dokument_A.pdf"
Peek()→ T
Vorderstes Element NUR ansehen — NICHT entfernen.
if (drucker.Count > 0)
    Console.WriteLine(drucker.Peek());
// → "Dokument_B.pdf" (bleibt da)
while-Verarbeitungsschleife
while (drucker.Count > 0)
{
    Console.Write(drucker.Dequeue() + " ");
}
// → A B C (FIFO!)
